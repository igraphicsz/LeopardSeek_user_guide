    获取持仓信息

    本页介绍获取用户持仓信息的方法以及获取到信息的格式以及意义。

    获取持仓信息

    使用 lzdc_broker 模块下 Broker 类中 get_position 方法获取用户持仓信息。

    # 调用方法时instrument是可选参数，如不传入则返回该用户所有持仓
position_dicts = self.broker.get_positions(instrument=self.instrument)

    引用必要的枚举

    leopardseek使用 python 枚举 来直观的表示用户持仓方向，该枚举也可以直接用于比较。

    获取持仓方向使用的 python 枚举（PosDirection） 与下单时使用的 python枚举（Direction） 不相同，请用户使用时注意区分。 

    from autotrader.comms.Enum import PosDirection # 引用枚举

"""
class PosDirection(Enum):
    Long = 2
    Short = 3
"""

    返回值格式

    get_position 返回值为列表（List），其中每个元素为字典，包括某个（多/空）方向仓位的详细信息。

    以下示例是 ao2605 的持仓情况，下文返回的列表表示该账户持有 ao2605 的一手多仓。

        [
        {
            'symbol': 'ao2605', 
            'direction': <PosDirection.Long: 2>, 
            'position': 1, 
            'frozen': 0, 
            'ydPosition': 0, 
            'ydFrozen': 0, 
            'tdPosition': 1, 
            'tdFrozen': 0, 
            'openPrice': 2816.0, 
            'positionProfit': 520.0, 
            'positionProfitRatio': None, 
            'updateTimestamp': 0
        }
    ]

    完整使用示例

    以下是示例策略中，关于获取持仓信息字典以及提取需要信息的使用示例。

    from autotrader.comms.Enum import PosDirection 

# ... generate signal ...

    # 获取持仓信息
    holding_direction = 0       # 初始化持仓方向
    holding_size = 0            # 初始化持仓手数
    current_profit = 0          # 初始化持仓收益
    position_dicts = self.broker.get_positions(self.instrument)

    # 对获取持仓信息的处理示例，仅考虑单方向持仓情况
    # （如果策略有锁仓的可能性，下面的处理不适用）
    if position_dicts:
        for position_dict in position_dicts:

            # 通过对枚举的比较判断，判断持仓的方向
            if position_dict["direction"] == PosDirection.Long:  # 多头
                current_profit = position_dict['positionProfit']
                holding_direction = 1
                holding_size = position_dict['position']

            elif position_dict["direction"] == PosDirection.Short:  # 空头
                current_profit = position_dict['positionProfit']
                holding_direction = -1
                holding_size = position_dict['position']

    可以通过遍历返回的仓位列表来判断持仓方向，持仓列表中的每个字典表示了该账户某个方向的仓位（若只有一个方向的仓位，如只有多仓或者只有空仓，则持仓列表中只有一个字典）。直接比较枚举来判断该仓位的方向，上述示例只适合于不存在锁仓情况的交易策略。
